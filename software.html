Artifact Overview

This enhancement focused on improving the modular architecture and maintainability of the HELIOS server. I restructured the code into services and routes using Express.js, introduced logging middleware, and ensured separation of concerns across modules.

Project Background

HELIOS is my custom-built self-hosted media and request management server running on an UNRAID-powered NAS system. I implemented this project to automate request handling, logging, and media indexing using services like Plex, Sonarr, and Radarr.

Enhancement Highlights

  • Restructured HELIOS request logger into modular Express routes and middleware
  • Improved scalability and maintainability by adopting single responsibility principles
  • Introduced centralized error handling and debug utilities to aid in future troubleshooting
  • Clean separation of frontend/backend behavior to isolate service responsibilities

Preview the Code

Below is a snippet from the refactored Express.js app used to handle requests and middleware.

const express = require("express");
const app = express();
const requestLogger = require("./middleware/requestLogger");

app.use(express.json());
app.use(requestLogger);

app.post("/log", (req, res) => {
  console.log(req.body);
  res.status(200).send("Logged successfully.");
});

app.listen(3000, () => console.log("Server running on port 3000"));
