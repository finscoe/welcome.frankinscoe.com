Artifact Overview

This enhancement focused on improving the modular architecture and maintainability of the HELIOS server. I restructured the code into services and routes using Express.js, introduced logging middleware, and ensured separation of concerns across modules.

Project Background

HELIOS is my custom-built self-hosted media and request management server running on an UNRAID-powered NAS system. I implemented this project to automate request handling, logging, and media indexing using services like Plex, Sonarr, and a custom-built HTTP interface.

Enhancement Highlights

  • Rewrote HELIOS request logger to split router logic into modular services and middleware
  • Improved scalability and testability by adopting object-oriented responsibility principles
  • Introduced external error tracking and debug logging to aid in future troubleshooting
  • Clean separation of frontend/backend behavior in isolated controller responsibilities

Preview the Code

Below is a snippet from the refactored Express.js app used to manage request services and middleware.


const express = require("express");
const app = express();
const requestQueue = require("./controllers/requestQueue");
const logger = require("./middleware/logger");

app.use(express.json());
app.use(logger);
app.post("/request", requestQueue);
app.listen(3000, () => {
  console.log("HELIOS server running...");
});
