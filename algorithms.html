Artifact Overview

This enhancement focused on developing and optimizing an algorithmic solution to manage media request processing in the HELIOS server project. The implemented Python-based queue system utilizes object-oriented design to manage asynchronous request handling.

Project Background

In HELIOS, users can submit requests for media downloads. This enhancement introduced a structured, class-based queue that ensures requests are processed in FIFO order while avoiding duplication and maintaining system performance.

Enhancement Highlights

  • Designed a robust Python queue class with enqueue, dequeue, and peek methods
  • Implemented a method to avoid duplicate requests and manage maximum queue length
  • Integrated real-time logging for each media request processed
  • Applied unit testing to validate the correctness of queue behavior

Preview the Code

Below is a code snippet from the core class used in the request queue system:

class MediaRequestQueue:
    def __init__(self, max_size=50):
        self.queue = []
        self.max_size = max_size

    def enqueue(self, request):
        if request not in self.queue and len(self.queue) < self.max_size:
            self.queue.append(request)

    def dequeue(self):
        if self.queue:
            return self.queue.pop(0)
        return None

    def peek(self):
        return self.queue[0] if self.queue else None
